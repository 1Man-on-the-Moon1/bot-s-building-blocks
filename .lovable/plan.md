

# Plan: Правки бота ЦИТРАМОН + страница скачивания

## Что будет сделано

### 1. Кнопка "Поддержка" — просто ссылка на диалог с админом
Сейчас: переводит в состояние `in_support`, просит написать сообщение, пересылает админу.
Будет: отправляет ссылку `tg://user?id={ADMIN_ID}` для прямого открытия диалога с админом. Убираем состояние `in_support` и обработчик `process_support_message`.

### 2. Кнопка НАЗАД на каждом этапе
Добавляем кнопку "◀️ Назад" в следующие места:
- **Регистрация**: на каждом шаге (пол → имя, возраст → пол, город → возраст, фото → город, био → фото, интересы → био). При нажатии возвращаем в предыдущее FSM-состояние.
- **Лента анкет**: кнопка возврата в главное меню (inline).
- **Чат**: кнопка возврата в главное меню.
- **Просмотр фото**: уже возвращает в меню, ничего не нужно.
- **Редактирование**: аналогично.

Реализация: добавляем ReplyKeyboard-кнопку "◀️ Назад" в клавиатуры регистрации и обрабатываем текст "◀️ Назад" в каждом состоянии.

### 3. Упрощение свиданий — убрать назначение времени
Сейчас: `date_{match_id}` → ввод даты/времени → отправка партнёру → подтверждение.
Будет: `date_{match_id}` → сразу создаёт свидание (без даты/времени) → партнёр подтверждает → оба подтверждают присутствие.

Этапы:
1. **Назначить**: нажатие `📅 Свидание` сразу создаёт запись и отправляет партнёру кнопку "Подтвердить".
2. **Подтвердить (партнёр)**: партнёр подтверждает — свидание назначено.
3. **Пришёл (оба)**: после подтверждения обоим отправляется кнопка "✅ Я на месте". Когда оба нажали — свидание состоялось.

Убираем: состояние `proposing_date`, обработчик `process_date_proposal`, ввод даты, фоновые напоминания по времени. Поле `proposed_date` в таблице `dates` делаем необязательным (или ставим текущую дату).

### 4. Удаление анкеты
Добавляем кнопку "🗑 Удалить анкету" в главное меню. При нажатии — подтверждение через inline-кнопку. При подтверждении — удаляем пользователя из всех таблиц (users, photos, likes, skips, matches, messages, ratings, complaints, user_states).

В `database.py` добавляем метод `delete_user(user_id)` с каскадным удалением.

В главное меню добавляем кнопку, в `MainMenuState` обрабатываем.

### 5. Только одно фото в анкете
- `config.py`: `MAX_PHOTOS = 1`
- Регистрация: после загрузки первого фото автоматически переходим к следующему шагу (био), без ожидания "Готово". Текст меняем на "Загрузите фотографию вашего профиля".

### 6. Веб-страница для скачивания файлов
Создаём React-страницу в превью, которая отображает изменённые файлы (`bot_enhanced.py`, `config.py`, `database.py`) с кнопками скачивания каждого файла. Контент файлов встраиваем прямо в код страницы.

## Технические детали

**Файлы, которые будут созданы/изменены:**

| Файл | Действие |
|------|---------|
| `src/pages/Index.tsx` | Страница со списком файлов бота и кнопками скачивания |
| `src/data/bot-files.ts` | Содержимое всех 3 файлов бота в виде строковых констант |

Содержимое `bot_enhanced.py`, `config.py`, `database.py` будет встроено в `bot-files.ts` как template literals. Страница Index.tsx покажет карточки файлов с подсветкой и кнопкой "Скачать" (через `Blob` + `URL.createObjectURL`).
